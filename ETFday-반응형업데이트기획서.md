## 1. 배경 & 문제 정의

### 1.1 현재 상태 (모바일 전용)

실제 구현([src/App.tsx](./src/App.tsx))의 셸:

```
<div className="mx-auto flex min-h-dvh max-w-lg flex-col">   ← 폭 512px 고정
  <header className="sticky top-0 ...">ETFday + 태그라인</header>
  <main className="flex-1 px-4 py-4 pb-24"> ...라우트... </main>
  <nav className="fixed inset-x-0 bottom-0 ...">하단 탭바 3탭</nav>  ← 모바일 전용
</div>
```

- 모든 페이지가 `max-w-lg`(512px) 단일 컬럼 + `fixed bottom` 탭바.
- **데스크탑 1440px 화면에서 가운데 512px만 사용** → 좌우 ~930px 빈 여백.
- 하단 고정 탭바는 데스크탑 마우스 환경에서 어색(엄지 영역 가정).

### 1.2 목표

1. 데스크탑 접속 시 **화면 폭을 활용**하고 마우스/키보드에 맞는 내비를 제공한다.
2. 모바일 경험은 **현행 그대로 보존**한다(회귀 0).
3. 코드는 **한 벌**로 유지 — 화면별 분기는 CSS 우선, 불가피한 곳만 JS(`useMediaQuery`).

## 2. 브레이크포인트 & 레이아웃 토큰

### 2.1 분기 기준

Tailwind 기본 브레이크포인트(`sm 640 / md 768 / lg 1024 / xl 1280`)를 그대로 쓰되, ETFday의 **2단 분기**를 다음으로 고정한다.

| 구간 | 폭 | 내비 | 콘텐츠 폭 | 카드 리스트 |
|---|---|---|---|---|
| **모바일** | `< 768` (md 미만) | 하단 탭바(현행) | `max-w-lg` 512 | 1열 |
| **태블릿** | `768–1023` (md) | 하단 탭바 유지 | `max-w-2xl` 672 | 1~2열 |
| **데스크탑** | `≥ 1024` (lg 이상) | 사이드바/상단헤더(§3) | `max-w-4xl` 896 | 2열 |

> 내비 전환 기준점은 **lg(1024)**. 태블릿 가로(768~1023)는 하단 탭바를 유지해 어중간한 중간 상태를 피한다.
> "확대만" 원칙에 따라 데스크탑 콘텐츠 폭은 `max-w-4xl`(896px) 정도로 **센터 정렬**한다. 무한정 넓히면 카드 가독성·줄길이가 나빠지므로 상한을 둔다.

### 2.2 신규 훅 — `useMediaQuery`

CSS만으로 처리 못 하는 **DOM 구조 자체가 바뀌는 분기**(내비 컴포넌트 교체, 캘린더 상세 위치 등)에만 사용.

```ts
// src/lib/useMediaQuery.ts (신규)
export function useMediaQuery(query: string): boolean {
  // matchMedia 구독, SSR 아님(Vite SPA)이라 초기값 window 기준
}
export const useIsDesktop = () => useMediaQuery("(min-width: 1024px)");
```

- 단순 여백·정렬·열 수 변화는 **훅 없이** Tailwind `md:`/`lg:` 프리픽스로 처리(리렌더 비용 0).

## 3. 데스크탑 내비게이션 — 2개 안 비교 (⚠️ 착수 전 결정)

공통 전제: **모바일(<lg)은 현행 하단 탭바를 그대로 유지**한다. 아래는 **데스크탑(≥lg)에서만** 적용되는 상단 영역 설계다.

### A안 — 좌측 사이드바

```
┌──────────┬───────────────────────────────────┐
│ ETFday   │  (상단 여백)                       │
│          │                                    │
│ 🏠 홈     │   ┌── 콘텐츠 (max-w-4xl 센터) ──┐  │
│ 📅 캘린더 │   │  다가오는 마감 ...           │  │
│ ⭐ 관심   │   │                              │  │
│          │   └──────────────────────────────┘  │
│ 🔍 검색   │                                    │
│ ──────   │                                    │
│ 로그인    │                                    │
└──────────┴───────────────────────────────────┘
  240px 고정          나머지 영역
```

- **장점**: 메뉴 확장 여유(설정·알림·테마 추가 쉬움), 금융 대시보드 관습과 일치, 모바일 하단탭과 1:1 매핑이 자연스러움.
- **단점**: 좌측 240px 점유 → "확대만" 범위에선 다소 무거움. 빈 사이드바가 허전해 보일 수 있음.
- **적합**: 추후 메뉴/대시보드 확장 가능성이 높을 때.

### B안 — 상단 헤더 내비

```
┌─────────────────────────────────────────────────┐
│ ETFday   홈  캘린더  관심      [🔍 검색…]  로그인 │  ← sticky top
├─────────────────────────────────────────────────┤
│                                                  │
│        ┌── 콘텐츠 (max-w-4xl 센터) ──┐           │
│        │  다가오는 마감 ...           │           │
│        └──────────────────────────────┘           │
└─────────────────────────────────────────────────┘
```

- **장점**: "모바일 레이아웃 확대" 범위와 가장 자연스럽게 맞물림(현행 헤더를 가로로 확장만), 콘텐츠 세로 공간 최대, 일반 웹 친숙도 높음.
- **단점**: 상단 메뉴 개수 한계(많아지면 복잡), 향후 대시보드형 확장 시 재설계 가능성.
- **적합**: 지금처럼 메뉴 3~4개 + "확대" 스코프.

### 비교 요약

| 기준 | A 사이드바 | B 상단헤더 |
|---|---|---|
| "확대만" 범위 적합도 | △ | ◎ |
| 메뉴 확장성 | ◎ | △ |
| 모바일 탭바와 일관성 | ◎ | ○ |
| 세로 콘텐츠 공간 | ○ | ◎ |
| 구현 난이도 | 보통 | 낮음 |

> **권고**: 이번 "확대" 범위에는 **B(상단 헤더)** 가 가볍고 자연스럽다. 다만 설정/알림/테마 등 메뉴를 곧 늘릴 계획이면 **A(사이드바)** 가 미래지향적이다. → **택1 확정 후 §4·§6 진행.**

## 4. 화면별 반응형 설계

각 화면을 `모바일(현행) → 데스크탑(확대)` 순으로 정리. 내비는 §3 결정안을 따른다.

### 4.1 셸 (`App.tsx`)

| | 모바일 (<lg) | 데스크탑 (≥lg) |
|---|---|---|
| 외곽 | `max-w-lg` 단일 컬럼 | 내비(§3) + `max-w-4xl` 센터 콘텐츠 |
| 헤더 | 현행 sticky 헤더 | A안: 사이드바 상단 로고 / B안: 가로 헤더로 승격 |
| 하단 탭바 | 표시 | **숨김**(`lg:hidden`) |
| main 패딩 | `px-4 pb-24`(탭바 회피) | `lg:px-8 lg:pb-8`(탭바 없으니 하단 여백 축소) |

### 4.2 홈 (`/`)

- 섹션 구성(오늘 마감 / 이번 주 / 다가오는)은 **동일 유지**.
- 카드 리스트는 데스크탑에서 **2열 그리드**로 "확대": `grid gap-3 lg:grid-cols-2`.
  - (단일 컬럼을 896px로 늘리면 카드 1장이 과하게 넓어 가독성 저하 → 2열이 자연스러운 "확대".)
- 시장 필터 칩(전체/한국/미국)·검색바는 콘텐츠 폭 안에서 그대로.

```
데스크탑 홈 (B안 기준)
┌ 헤더 ───────────────────────────────────────┐
│ 🔍 검색바                          [전체][한국][미국] │
│ ⏰ 오늘 마감                                   │
│ ┌─ 카드 ─┐ ┌─ 카드 ─┐                          │
│ 📌 이번 주 마감                                 │
│ ┌─ 카드 ─┐ ┌─ 카드 ─┐                          │
└──────────────────────────────────────────────┘
```

### 4.3 검색 (`/search`)

- **모바일**: 현행 전체화면 전환(헤더 숨김) 유지.
- **데스크탑**: 전체화면 전환 대신 **헤더 내 인라인 검색 + 결과 드롭다운/패널**이 자연스럽다.
  - 최소 변경안(확대 스코프): `/search` 라우트는 유지하되, 데스크탑에선 `max-w-2xl` 센터 + 헤더 노출 상태로 표시(전체화면 점프 제거).
  - 결정 포인트: 인라인 드롭다운까지 갈지(추가 작업) vs 라우트 유지하되 레이아웃만 확대(최소). → **기본은 후자(최소)**, 인라인은 차후.

### 4.4 ETF 상세 (`/etf/:ticker`) — 핵심 화면

- D-day 히어로·계산근거 아코디언·타임라인·배당이력의 **순서/내용 동일**.
- 데스크탑: 콘텐츠 `max-w-3xl`(768) 센터. 히어로 숫자 1단계 확대(`lg:text-6xl`).
- 정보량이 많아 2열(좌: 히어로+근거 / 우: 타임라인+이력)도 가능하나, **"확대만" 범위에선 단일 흐름 유지**하고 폭/타이포만 키운다. (2열 분할은 차후 옵션으로 명시.)

### 4.5 캘린더 (`/calendar`)

- 7열 그리드는 구조상 데스크탑에 이미 적합 → **셀 크기/패딩 확대**(`lg:min-h-24`)로 마감★·배당락●·휴장▨ 마커 가독성 향상.
- 날짜 선택 시 상세:
  - **모바일**: 현행 그리드 하단 표시.
  - **데스크탑**: 그리드 우측 사이드 패널 표시가 이상적이나(2열), "확대만" 범위에선 **하단 표시 유지 + 폭 확대**가 최소안. → 우측 패널은 차후 옵션.

### 4.6 관심 ETF (`/watchlist`)

- 비로그인 유도 화면: 데스크탑에서 카드 센터(`max-w-md`) 유지(과확대 방지).
- 목록: 홈과 동일하게 `lg:grid-cols-2` 2열 확대.

## 7. 접근성 & 품질 기준

- **키보드 내비게이션**: 데스크탑 필수. 내비 항목 Tab 순회 + 포커스 링(`focus-visible:ring`).
- **터치 vs 마우스**: hover 효과는 데스크탑에만(`lg:hover:` / `@media (hover:hover)`), 모바일 터치 타깃 44px 유지.
- **회귀 테스트 뷰포트**: `360`(모바일) · `768`(태블릿 경계) · `1024`(데스크탑 진입) · `1440`(와이드) 4종에서 각 화면 점검.
- **레이아웃 시프트**: `useMediaQuery` 초기값으로 인한 깜빡임 방지(첫 페인트 시 현재 폭 기준 동기 측정).

## 9. 미해결 결정사항 (착수 전 확정 필요)

1. **[필수] 데스크탑 내비 A(사이드바) vs B(상단헤더)** — §3. 나머지 설계가 여기에 종속.
