# ETFday 실행 가이드

## 사전 준비 (최초 1회)

```bash
# Bun 설치 (Gencow CLI 필수 런타임)
curl -fsSL https://bun.sh/install | bash
```

## 백엔드 실행

```bash
cd ~/myproject/ETFday/ETFday

npx gencow login      # 브라우저로 Gencow Cloud 로그인 (최초 1회)
npx gencow dev        # 백엔드 서버 시작 → http://localhost:5456
                      # Admin Dashboard → http://localhost:5456/_admin
```

`gencow dev`가 스키마를 클라우드 DB에 반영하고 `src/gencow/api.ts`를 자동 갱신합니다.

## 시드 데이터 입력 (최초 1회)

```bash
npx gencow db:seed    # gencow/seed.ts 실행 — 시장·휴장일·인기 ETF 8종
```

또는 Admin Dashboard에서 `seed.run` mutation 실행.

## 프론트엔드 실행

```bash
npm run dev:web       # Vite 개발 서버 → http://localhost:5173
```

`.env`의 `VITE_API_URL`이 비어 있으면 `http://localhost:5456`(로컬 백엔드)으로 연결됩니다.

## 자주 쓰는 명령

| 명령 | 설명 |
|---|---|
| `npx gencow dev` | 백엔드 dev 서버 (hot-reload + codegen) |
| `npm run dev:web` | 프론트 dev 서버 |
| `npx gencow db:push` | 스키마 변경 즉시 반영 |
| `npm run typecheck` | 전체 타입체크 |
| `bun gencow/lib/deadline.test.ts` | 매수마감 계산 테스트 (13 케이스) |
| `npm run build` | 타입체크 + 프론트 빌드 |
| `npx gencow deploy --static dist/` | 백엔드+프론트 클라우드 배포 |

## 환경변수 (운영)

`.env` 또는 `npx gencow secret set` 으로 클라우드에 주입:

| 변수 | 용도 | 미설정 시 동작 |
|---|---|---|
| `FMP_API_KEY` | 미국 배당 확정 데이터 (⚠️ 무료 플랜은 ETF 402 차단 — 유료 필요) | **Yahoo 폴백**(키 불필요, 비공식)으로 자동 동작 |
| `RESEND_API_KEY` | 이메일 알림 발송 (resend.com) | `alerts.sendDaily`가 dry-run (로그만) |
| `ALERT_FROM_EMAIL` | 발신 주소 (Resend에서 도메인 인증 필요) | `alerts@etfday.app` (placeholder) |
| `APP_URL` | 이메일 본문의 앱 링크 | `https://etfday.app` |
